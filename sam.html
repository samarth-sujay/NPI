Data Source: Summary
1. Classiﬁcation of data into master and transactional
a. Master
i. The data which is largely static like state list, district list, ministry list
b. Transactional
i. The data which changes quite often like Lok Sabha/Rajya Sabha members, who’who,
judges etc
2. Conﬂicting masters
a. Which master IGOD or S3WAAS to refer for state and district list ?
Design Approach
First Ingest Master Data
Resolve conﬂicts between Masters
Link to ingested masters while ingesting transactional data
